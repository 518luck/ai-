# 01 - AI开发全景与趋势

## 三代演进

| 时代 | 时间 | 核心方式 | 开发者角色 |
|------|------|---------|-----------|
| 第一代：Tab补全 | 2023-2024 | 自动补全代码 | 逐行编写 |
| 第二代：同步Agent | 2025 | 对话式prompt-response | 逐步指导 |
| 第三代：云Agent | 2026 | 自主并行执行 | 定义问题+审查结果 |

## 主要工具/平台

| 工具 | 定位 | 特点 |
|------|------|------|
| Cursor | AI原生IDE | Agent模式、云Agent并行、自动化触发 |
| GitHub Copilot | IDE插件 | 代码补全、PR审查、Issue分配Agent |
| Claude Code (Agent SDK) | 可编程Agent框架 | 自定义Agent、子Agent、Hook系统 |
| OpenAI Codex | 终端/云端Agent | 开源CLI、Rust编写 |
| Windsurf | AI IDE | Cascade工作流 |

## 关键趋势数据

- 2025年3月：Tab用户是Agent用户的2.5倍
- 2026年2月：Agent用户是Tab用户的2倍
- 一年内Agent使用量增长15倍
- "大多数Cursor用户已经不再按Tab键了"
- 35%的PR由云Agent自主创建（Cursor内部数据）

## AI开发的核心转变

```
传统：工程师写代码 → 测试 → 部署
现在：工程师定义问题 → Agent写代码 → AI审查 → 自动合并/人工审查
```

工程师角色从"写代码的人"变成"指挥Agent工厂的人"。

## 适用场景判断

| 适合AI开发 | 不适合AI开发 |
|-----------|-------------|
| Web应用/SaaS | 嵌入式系统 |
| 内部工具/管理后台 | 金融核心交易系统 |
| 快速原型/MVP | 医疗设备控制软件 |
| 有完善测试覆盖的项目 | 航空/军事关键系统 |
| 能快速回滚的部署环境 | 无法灰度发布的环境 |

**判断标准：能快速试错 + 能回滚 = 适合AI开发**
