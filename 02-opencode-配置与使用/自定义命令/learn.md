---

分析本次会话，并提取出非显而易见的知识（learnings）添加到对应的 `AGENTS.md` 文件中。

`AGENTS.md` 文件可以存在于任何层级的目录下，不仅限于项目根目录。当 agent 读取某个文件时，其父级目录中的任何 `AGENTS.md` 都会自动加载到读取工具的上下文中。因此，请将这些知识尽可能放在与相关代码最接近的目录中：

- 项目全局级别的知识 → 根目录下的 `AGENTS.md`
- 特定包/模块级别的知识 → `packages/foo/AGENTS.md`
- 特定功能级别的知识 → `src/auth/AGENTS.md`

什么样内容可以算作知识（**仅限**非显而易见的发现）：

- 文件或模块之间隐藏的关联关系
- 与代码字面表现不一致的实际执行路径
- 非直观的配置项、环境变量或标志（flags）
- 在错误信息具有误导性时的调试突破口
- API 或工具的某些特殊脾性及规避方案（workarounds）
- README 中未记载的构建/测试命令
- 架构决策与设计约束
- 必须协同修改的文件

**不**应包含的内容：

- 官方文档中显而易见的事实
- 语言或框架的常规标准行为
- 已经在某个 `AGENTS.md` 中记录过的内容
- 冗长的解释
- 特定于该会话的细节信息

步骤流程：

1. 审查会话以寻找新发现、耗费多次尝试才解决的错误、以及意料之外的关联。
2. 确定作用域 — 每项知识适用于哪一级目录？
3. 读取相关层级下已有的 `AGENTS.md` 文件。
4. 在合适的层级创建或更新 `AGENTS.md`。
5. 保持每条洞察记录控制在 1-3 行以内。

更新完成后，总结创建或更新了哪些 `AGENTS.md` 文件，以及每个文件中分别记录了多少条知识点。
