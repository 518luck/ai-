## 一、Amplitude（数据分析公司）— 最先进的自主开发流水线

### 完整流水线：Slack → Agent → Ticket → PR → 自动合并

```
客户在Slack反馈Bug/需求
        ↓
云Agent自动触发（Cursor Automation）
        ↓
Agent检查Linear是否已有工单
  ├── 有 → 补充客户上下文
  └── 没有 → 探索代码库、创建新工单、直接开PR修复
        ↓
Bugbot自动代码审查
  ├── 低风险PR → 自动合并到生产（60-70%的PR）
  └── 高风险PR → 自动路由给对应工程师审查
```

### 每小时自动运行的遗留代码迁移

- 定时Agent扫描CSS文件，自动替换为Tailwind类
- 自动处理20,000+个遗留React布局组件
- 替换完成后自动开PR、发Slack通知

### 关键数据

| 指标 | 数值 |
|------|------|
| 每周自动Agent运行 | 1000+次，无需人工触发 |
| 低风险PR直接合并率 | 60-70% |
| 生产提交增长 | 3倍 |
| Cursor在代码库中的贡献排名 | Top 3（按commit量） |

### 工程师日常工作模式

- 新想法：在Slack讨论，直接从Slack线程启动云Agent
- 详细迭代：拉回本地Agent精细调整
- 云Agent并行：同时运行多个Agent，每个有独立虚拟机和完整开发环境

|------|---------|--------------|
| 开发周期 | 4个月 | 3周 |
| 效率提升 | - | 约5倍 |

## 三、PayPal（8000名开发者）— 大型企业推广策略

### 推广路径

1. 先在最高影响团队试点（2周内从双周部署变为每日部署）
2. 自然传播 — 工程师看到同事效率提升后自发采用
3. 高采用率团队 — 采用率超90%，部署频率显著提升

### 工作流程变革

```
传统模式（已淘汰）：
设计文档(PRD) → 工程师编码 → 构建 → 部署（线性流程）

AI模式（当前）：
产品经理带可运行原型找工程师
        ↕
工程师用自己的原型回应
        ↓
数小时内从想法到可运行原型
        ↓
快速迭代 → 每日多次部署
```

### 关键成果

- Java升级3000个应用：从8-12个月缩短到2个月（快6倍）
- 交付能力提升40%（2026年预计比2025年多交付40%的功能）
- 部署节奏：从每月一次 → 每天多次
- 变更失败率下降

### 度量理念（非常重要）

> "如果你度量AI生成代码的比例，开发者就会让AI写冗余的函数。" — PayPal开发者平台负责人

PayPal刻意**避免**度量AI代码占比，只关注：
- 部署频率
- 从想法到生产的交付周期
- 变更失败率

## 五、Cursor自身 — AI开发的最前沿实践

### Cursor团队当前工作方式

- 35%的PR由云Agent自主创建（无需人工干预）
- 工程师几乎100%的代码由Agent编写
- 工程师时间分配：
  - 分解问题（定义任务）
  - 审查产物（视频演示、日志、预览，而非diff）
  - 给出反馈
- 同时启动多个Agent并行工作，而非手把手指导一个

## 六、Rippling（HR SaaS公司）— 个人AI助理

```
工程师把笔记/Todo/Loom链接扔进Slack
→ cron Agent每2小时读一遍 + GitHub PR + Jira + Slack mentions
→ 自动去重、归类
→ 生成个人仪表板发到Slack
```

实际使用的自动化Agent：
- Slack触发的Jira工单创建
- Confluence文档自动摘要
- 事故分类和响应
- 每周状态报告
- on-call交接摘要
- 好用的Agent会在团队间分享

## 总结：企业AI开发的标准工作流模板

```
┌─────────────────────────────────────────────────┐
│               触发层（Trigger）                    │
│  Slack消息 / 定时任务(Cron) / PR创建 / Jira工单   │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│               Agent层（Execution）                │
│  云Agent自主运行：探索代码 → 编写代码 → 测试验证    │
│  （可多个并行，各有独立VM和完整开发环境）             │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│               审查层（Review）                     │
│  AI代码审查（Bugbot）自动分级风险                   │
│  低风险 → 自动合并    高风险 → 路由给工程师          │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│               反馈层（Artifact）                   │
│  视频演示 / 实时预览 / 日志（取代传统diff）          │
└─────────────────────────────────────────────────┘
```
