# 多Agent协作系统（前沿研究）

## 背景

Cursor与NVIDIA合作的研究项目：用多Agent系统优化GPU内核。

## 为什么需要多Agent

单一Agent在面对开放性问题时表现不佳，因为AI模型最擅长处理训练数据中常见的**狭窄任务**。

多Agent系统可以：
- 将复杂问题分解为多个子任务
- 各Agent并行处理自己擅长的部分
- 协调者动态重新分配工作负载

## 架构

```
┌──────────────────────────────────────┐
│        Planner Agent（规划者）          │
│  分配任务、监控进度、重新平衡工作负载      │
└──────┬──────────┬──────────┬─────────┘
       ↓          ↓          ↓
┌──────────┐┌──────────┐┌──────────┐
│ Worker 1 ││ Worker 2 ││ Worker 3 │
│ 独立执行  ││ 独立执行  ││ 独立执行  │
│ 优化任务A ││ 优化任务B ││ 优化任务C │
└──────────┘└──────────┘└──────────┘
```

## 协调协议

**整个协调协议写在一个markdown文件中**，指定：
- 输出格式
- 规则
- 测试标准

## 自主循环

Worker自主学会了：
1. 调用benchmarking pipeline
2. 分析性能数据
3. 定位瓶颈
4. 应用优化
5. 重新测试

形成 **测试→调试→优化** 的自主循环，无需人类干预。

## 两种执行模式

| 模式 | 说明 |
|------|------|
| CUDA C + inline PTX | 最低级硬件访问 |
| CuTe DSL | 高层抽象，测试学习新API的能力 |

## 结果

| 指标 | 数值 |
|------|------|
| 优化问题总数 | 235个GPU内核优化 |
| 平均加速 | 38%（geomean） |
| 超过2x加速的占比 | 19% |
| 超过基线的比例 | 149/235（63%） |
| 硬件 | 27块NVIDIA Blackwell 200 GPU |
| 运行时长 | 3周 |
| attention问题 | SOL得分0.9722，84%加速 |
| GEMM内核 | 达到cuBLAS手写优化的86% |

## 关键洞察

> 单Agent在训练分布内的狭窄任务上表现最好。多Agent系统可以在**训练分布之外**的全新问题上取得突破。

这意味着多Agent系统是解决AI"没见过的问题"的一条路径。
