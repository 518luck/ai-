# Bootstrapping Agent（自我进化Agent）

## 是什么

Cursor的训练流水线：用旧版本Agent为新版本Agent创建训练环境，实现自我进化。

## 两阶段流水线

### 阶段1：目标设定Agent

```
输入：代码库
行为：
  1. 探索代码库结构
  2. 提出10个关键命令
  3. 为每个命令定义期望输出
  4. 建立"正确环境配置"的标准
输出：目标规格文档
```

### 阶段2：执行Agent

```
输入：初始状态 + 3个目标命令
行为：
  1. 探索代码库
  2. 配置开发环境
  3. 测试是否达到目标
  4. 失败则重试（最多5次）
输出：配置好的开发环境
```

## Agent的自主策略

执行Agent会用各种手段达标：

- Mock缺失的文件
- 创建占位图片
- 假数据库表
- 安装额外组件（S3文件夹、sidecar容器）
- 创建MinIO配置或Docker容器
- 创建长期运行进程的启动脚本
- 搜索网络查找项目文档

## 自我进化机制

```
Composer 1.5（旧模型）创建环境
        ↓
用这些环境训练 Composer 2（新模型）
        ↓
Composer 2 得分：61.7% vs Composer 1.5 的 47.9%
        ↓
Composer 2 为下一代模型提供更好的基础
        ↓
循环...
```

## 意义

这是AI系统**用自己的能力为自己创造更好的训练数据**的实例。

每一代Agent：
1. 能力更强
2. 能处理更复杂的环境
3. 为下一代提供更高的起点

这不是理论，是Cursor已经在生产中使用的流程。
