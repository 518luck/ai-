# 通用规则清单

> 从"杂乱的规则"和 4 个真实项目中提炼的通用 AI 编程约束规则。

## 一、代码风格（TypeScript）

| 规则 | 说明 | 来源 |
|------|------|------|
| 避免 `any` | 所有新代码用 TypeScript，依赖类型推断 | Kibana, opencode |
| 显式返回类型 | 公共 API 和导出函数必须声明返回类型 | Kibana |
| `import type` | 纯类型导入用 `import type` | Kibana, opencode |
| 不压制类型错误 | 禁止 `@ts-ignore` / `@ts-expect-error`，修根因 | Kibana, wp-calypso |
| 函数组件优先 | React 组件必须用函数式 + 显式 props 类型 | Kibana, wp-calypso |
| 优先 `const` | 用 const + 三元/early return 替代 let 赋值 | opencode |
| 避免 `else` | 用 early return 替代 else 分支 | opencode |
| 减少变量 | 值只用一次就内联，减少中间变量 | opencode |
| 描述性命名 | 避免（紧凑循环外的）单字母名 | Kibana |

## 二、代码风格（Python）

| 规则 | 说明 | 来源 |
|------|------|------|
| 避免 `Any` | 优先具体类型 | Home Assistant |
| 类型标注 | 测试函数参数必须有类型标注 | Home Assistant |
| 注释解释"为什么" | 不写"做什么"的注释 | Sentry |
| 输入用抽象类型 | 参数用 `Sequence`，返回值用 `list` | Sentry |

## 三、控制流与错误处理

| 规则 | 说明 | 来源 |
|------|------|------|
| 优先 early return | 避免深层嵌套 | Kibana, opencode |
| 显式处理错误 | API 返回类型化错误 | Kibana |
| 避免 `try`/`catch` | 优先用 Effect schema helper 或其他方式 | opencode |
| 避免空 except | 只捕获可处理的异常 | Sentry |

## 四、测试

| 规则 | 说明 | 来源 |
|------|------|------|
| 测实际实现 | 避免 mock，不重复逻辑到测试 | opencode, Kibana |
| 不删/跳/注释测试 | 测试失败修根因 | Kibana |
| 测试文件镜像源码 | `src/foo/bar.py` → `tests/foo/test_bar.py` | Sentry |
| 禁止 wall-clock 等待 | 用 readiness signal / Deferred / Latch | opencode |
| 用工厂方法 | 禁止 `Model.objects.create`，用 `self.create_*` | Sentry |
| 必须用 pytest | 禁止 `unittest` 风格 | Sentry |

## 五、贡献规范

| 规则 | 说明 | 来源 |
|------|------|------|
| 修根因 | 不做临时修补 | Kibana |
| 聚焦变更 | 不做无关重构 | Kibana |
| 更新文档和测试 | 行为变化时同步更新 | Kibana |
| PR 以 draft 开始 | 所有 PR 先以 draft 开启 | Kibana, wp-calypso |
| 前后端分 PR | 前端和后端变更分开提交 | Sentry |
| 不含客户信息 | PR/commit 中禁止包含真实客户数据 | Sentry |
| Commit 规范 | `type(scope): summary` 格式 | opencode |

## 六、README 阅读规则

在编辑任何子目录中的文件之前，请阅读从仓库根目录到目标文件所在目录路径上的所有 README.md 文件。

**示例**：编辑 `turbopack/crates/turbopack-ecmascript-runtime/js/src/nodejs/runtime/runtime-base.ts` 之前，阅读：
- `turbopack/README.md`
- `turbopack/crates/README.md`
- `turbopack/crates/turbopack-ecmascript-runtime/README.md`
- `turbopack/crates/turbopack-ecmascript-runtime/js/README.md`

## 七、Vibe Coding 实用提示词

Anthropic 研究员 Thariq 分享的日常开发提示词，解决 vibe coding 上下文管理黑盒问题：

> implement \<SPEC> and while you do, keep a running implementation-notes.html file (or markdown) with decisions you had to make weren't in the spec, things you had to change, tradeoffs you had to make or anything else I should know

核心思路：让 agent 构建外挂式记事本，对开发过程中的重要信息留痕与提醒。
