本文件为 AI agent 在 Grafana 仓库中处理代码时提供指导。

**针对特定目录的 agent 文件用于专门领域：在这些目录下工作时，请阅读对应文件：**

- `docs/AGENTS.md`：文档风格指南，适用于 `docs/` 下的工作
- `public/app/features/alerting/unified/AGENTS.md`：Alerting squad 的开发模式

## Project Overview

Grafana 是一个监控和可观测性平台。后端使用 Go，前端使用 TypeScript/React。该仓库是一个 monorepo，前端使用 Yarn workspaces，后端使用 Go workspaces。

## Principles

- 遵循周围代码已有的模式
- 为新功能编写测试
- 保持改动聚焦，避免过度工程化
- 前端和后端改动使用独立 PR，因为它们的部署节奏不同
- 安全：防止 XSS、SQL injection、command injection

## Comments

- 只有当注释能解释某件事**为什么**这样做，或揭示读者为了安全修改代码必须了解的非显而易见逻辑时，才添加注释。如果代码本身已经足够清晰，就不需要注释。
- 不要在代码注释中包含链接，例如 Slack、GitHub、Jira 等。

## Human Review Gates

运行 `git push` 之前，必须停下来并获得明确的人类批准。改动准备好后，展示改动摘要并等待指示。任务描述里的 "Open a PR" 表示意图，不等于允许在未经 review 的情况下 push。

## Commands

### Build & Run

```bash
make run                          # 带热重载的后端，localhost:3000，admin/admin
make build-backend                # 仅构建后端
yarn start                        # 前端开发服务器，监听文件变更
yarn build                        # 前端生产构建
```

### Test

```bash
# Backend
go test -run TestName ./pkg/services/myservice/   # 指定测试
make test-go-unit                                  # 所有单元测试
make test-go-integration                           # 集成测试

# Frontend
yarn test path/to/file                             # 指定文件
yarn test -t "pattern"                             # 按名称模式筛选
yarn test -u                                       # 更新快照

# E2E
yarn e2e:playwright path/to/test.spec.ts           # 指定测试
```

### Lint & Format

```bash
make lint-go                      # Go linter
yarn lint                         # ESLint
yarn lint:fix                     # ESLint 自动修复
yarn prettier:write               # Prettier 自动格式化
yarn typecheck                    # TypeScript 检查
```

### Code Generation

```bash
make gen-go                       # Wire DI，修改 service init 后运行
make gen-cue                      # CUE schemas，修改 kinds/ 后运行
make gen-apps                     # App SDK apps
make swagger-gen                  # OpenAPI/Swagger specs
make gen-feature-toggles          # Feature flags，pkg/services/featuremgmt/
make i18n-extract                 # i18n strings
make update-workspace             # Go workspace，添加模块后运行
```

### Dev Environment

```bash
yarn install --immutable                          # 安装前端依赖
make devenv sources=postgres,influxdb,loki        # 启动后端依赖服务
make devenv-down                                  # 停止后端依赖服务
make lefthook-install                             # Pre-commit hooks
```

## Architecture

### Backend (`pkg/`)

| Directory | Purpose |
| --- | --- |
| `pkg/api/` | HTTP API handlers 和 routes |
| `pkg/services/` | 按领域划分的业务逻辑，例如 alerting、dashboards、auth 等 |
| `pkg/server/` | 服务器初始化和 Wire DI 设置，见 `wire.go` |
| `pkg/tsdb/` | 时序数据库查询后端 |
| `pkg/plugins/` | 插件系统和加载器 |
| `pkg/infra/` | 日志、指标、数据库访问 |
| `pkg/middleware/` | HTTP middleware |
| `pkg/setting/` | 配置管理 |

**Patterns**：Wire DI，修改后用 `make gen-go` 重新生成；services 在同一个 package 中实现 interfaces；业务逻辑放在 `pkg/services/<domain>/`，不要放在 API handlers 中；数据库访问通过 `sqlstore`；插件通信通过 gRPC/protobuf。

### Frontend (`public/app/`)

| Directory | Purpose |
| --- | --- |
| `public/app/core/` | 共享服务、组件、工具函数 |
| `public/app/features/` | 按领域划分的功能代码，例如 dashboard、alerting、explore |
| `public/app/plugins/` | 内置插件，其中许多是 Yarn workspaces |
| `public/app/types/` | TypeScript 类型定义 |
| `public/app/store/` | Redux store 配置 |

**Patterns**：使用基于 slices 的 Redux Toolkit，不使用旧 Redux；使用带 hooks 的函数组件；通过 `useStyles2` 使用 Emotion CSS-in-JS；使用 RTK Query 获取数据；使用 React Testing Library 编写测试。

### Shared Packages (`packages/`)

`@grafana/data`（数据结构）、`@grafana/ui`（组件）、`@grafana/runtime`（运行时服务）、`@grafana/schema`（由 CUE 生成的类型）、`@grafana/scenes`（dashboard framework）。

### Backend Apps (`apps/`)

使用 Grafana App SDK 的独立 Go apps：`apps/dashboard/`、`apps/folder/`、`apps/alerting/`。

### Plugin Workspaces

这些内置插件需要单独的构建步骤：`azuremonitor`、`cloud-monitoring`、`grafana-postgresql-datasource`、`loki`、`tempo`、`jaeger`、`mysql`、`parca`、`grafana-pyroscope-datasource`、`grafana-testdata-datasource`。

构建指定插件：`yarn workspace @grafana-plugins/<name> dev`

## Key Notes

- **Wire DI**：后端 service init 变更后需要运行 `make gen-go`。Wire 会在编译时发现循环依赖。
- **CUE schemas**：`kinds/` 中的 dashboard/panel schemas 会通过 `make gen-cue` 同时生成 Go 和 TS 代码。
- **Feature toggles**：定义在 `pkg/services/featuremgmt/` 中，并自动生成代码。变更后运行 `make gen-feature-toggles`。
- **Go workspace**：定义在 `go.work` 中。添加 Go modules 后运行 `make update-workspace`。
- **Build tags**：`oss`（默认）、`enterprise`、`pro`。
- **Config**：默认配置在 `conf/defaults.ini`，覆盖配置在 `conf/custom.ini`。
- **Database migrations**：位于 `pkg/services/sqlstore/migrations/`。测试时先运行 `make devenv sources=postgres_tests,mysql_tests`，再运行 `make test-go-integration-postgres`。
- **CI sharding**：后端测试使用 `SHARD` / `SHARDS` 环境变量进行并行化。

## Cursor Cloud specific instructions

### Prerequisites

- **Node.js v24.x**，确切版本见 `.nvmrc`。使用 `nvm install` / `nvm use` 保持一致。
- **Go 1.25.7**，见 `go.mod`。VM 中已预装。
- **Yarn 4.11.0**，通过 corepack 使用，已打包在 `.yarn/releases/` 中。如果找不到 `yarn`，运行 `corepack enable`。
- **GCC**，后端的 CGo/SQLite 编译需要它。

### Running services

- **Backend**：`make run` 会构建并启动带热重载（air）的 Grafana 后端，地址为 `localhost:3000`。默认登录信息是 `admin` / `admin`。由于 debug symbols（`-gcflags all=-N -l`），首次构建约需 3 分钟；后续热重载重建会更快。
- **Frontend**：`yarn start` 会启动 webpack 开发服务器并监听变更。后端会代理到该服务器。首次编译约需 45 秒。
- 不需要外部数据库，Grafana 默认使用嵌入式 SQLite。

### Testing gotchas

- **Frontend tests**：`yarn test` 脚本默认包含 `--watch`。始终使用 `yarn jest --no-watch`，或添加 `--watchAll=false`，确保测试只运行一次并退出。
- **Backend tests**：由于依赖图较大，某些 package（例如 `pkg/api/`）的测试编译较慢，约需 2 分钟。尽可能使用 `-run TestName` 进行定向测试。
- 所有标准 build/test/lint 命令都记录在上面的 Commands 部分。
