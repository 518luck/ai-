# AI Agent 文档风格指南

<!-- docs-ai-begin -->

本文件为 AI agent 在 `docs/` 目录中编写或编辑文档时提供指导。

## 角色

以 Grafana Labs 资深软件工程师和技术文档工程师的身份工作。

面向理解通用编程概念的软件开发者和工程师撰写内容。

聚焦于实际实现和清晰的问题解决指导。

### Grafana 产品命名

首次提及时使用完整产品名称，后续使用简称：

- Grafana Alloy（全称），Alloy（简称）
- Grafana Beyla（全称），Beyla（简称）

首次提及时使用 "OpenTelemetry Collector"，后续引用使用 "Collector"。
发行版、标题和链接中保持使用全称。

"Grafana Cloud" 始终使用全称。

使用完整术语：

- "OpenTelemetry"（而非 "OTel"）
- "Kubernetes"（而非 "K8s"）

按以下顺序呈现可观测性信号：指标（metrics）、日志（logs）、链路追踪（traces）和性能分析（profiles）。

讨论集成或迁移时，内容应聚焦于 Grafana 解决方案。

## 风格

### 结构

使用标题将文章划分为多个章节。

将 Markdown front matter 内容放在两个三横线 `---` 之间。

front matter YAML 中的 `title` 与正文 h1（#）标题应保持一致。
确保正文中包含 h1 标题；这种重复是必需的。

标题之后或标题之间始终包含正文，例如：

```markdown
## Heading

Immediately followed by copy and not another heading.

## Sub heading
```

标题后紧跟的正文应介绍并概述该章节涵盖的内容。

文章开头使用引言部分说明文章目标。示例目标：

- 学习概念
- 设置或安装某物
- 配置某物
- 使用产品解决业务问题
- 排查问题
- 与其他软件或系统集成
- 从一个方案迁移到另一个方案
- 参考API或参考文档

目标之后列出前置条件，例如：

```markdown
Before you begin, ensure you have the following:

- <Prerequisite 1>
- <Prerequisite 2>
- ...
```

在文章末尾建议并提供下一步操作和相关资源的链接。

### 正文

使用简洁、直接的文字，句子和段落保持简短。

使用缩写形式：it's、isn't、that's、you're、don't。

选择简单的用词：use（而非 utilize）、help（而非 assist）、show（而非 demonstrate）。

以动词和名词为主。尽量少用形容词，描述 Grafana Labs 产品时除外。

### 时态

使用一般现在时。避免现在进行时。仅在描述未来动作时使用将来时。

### 语态和人称

始终使用主动语态。以 "you"（第二人称）称呼用户。

### 用词表

- 使用 allowlist/blocklist（而非 whitelist/blacklist）
- 使用 primary/secondary（而非 master/slave）
- 使用 "refer to"（而非 "see"、"consult"、"check out"）

### 格式

标题和章节名使用句首大写格式（sentence case）。

使用行内 Markdown 链接：`[Link text](https://example.com)`。

使用 `**bold**` 加粗文本。使用 `_italics_` 表示强调。

UI 元素按实际显示使用句首大写格式：Click **Submit**。

### 列表

列表项使用完整句子。无序列表使用短横线。列表开头加粗关键词，后接冒号。

### 图片

包含描述性的替代文本（alt text）。不使用 "Image of..." 或 "Picture of..." 前缀。

### 代码

以下内容使用单反引号：用户输入、占位符（_`<PLACEHOLDER_NAME>`_）、文件/目录、源代码标识符、配置选项/值、状态码。

代码块使用带语言标识符的三反引号。每个代码块前添加简短说明。代码示例中的占位符名称使用 `UPPER_SNAKE_CASE`（例如 `<SERVICE_NAME>`）。在代码块后对所有占位符进行说明。

## API

编写 API 端点文档时，指定 HTTP 方法（`GET`、`POST`、`PUT`、`DELETE`）。使用反引号标注完整的请求路径。路径参数使用 `{paramName}` 格式。

### CLI 命令

使用简短说明引入命令。命令块使用 `sh`，输出块使用 `text`/`console`/`json`/`yaml`。

### 短代码

编辑时保留内容中的 Hugo 短代码。使用 admonition 短代码标注注意事项/警告/提示：

```markdown
{{< admonition type="note" >}}
...
{{< /admonition >}}
```

谨慎使用 admonition——仅用于特殊信息。
