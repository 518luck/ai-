# 测试指南

本文档旨在解答单元测试中最常见的"如何做"类问题。

## 测试的推荐实践

- 测试组件时，默认使用 `*ByRole` 查询，因为这有助于在测试中关注可访问性。
- 也可以使用 `*ByLabelText` 查询来测试组件。不过，我们推荐 `*ByRole` 查询，因为它们[更健壮](https://testing-library.com/docs/queries/bylabeltext/#name)。

## 测试用户交互

我们使用 [user-event](https://testing-library.com/docs/user-event/intro) 库来模拟测试中的用户交互。该库优于内置的 `fireEvent` 方法，因为它更准确地反映用户与元素的真实交互方式。

使用 `userEvent` 时有两个重要注意事项：

1. `userEvent` 中的所有方法都是异步的，因此调用时需要使用 `await`。
1. 直接调用 `userEvent` 的方法在未来版本中可能不再支持。因此，需要在测试前先调用 `userEvent.setup()`。该方法返回一个包含所有方法的 `userEvent` 实例。可以通过工具函数简化此设置过程：

```tsx
import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";

function setup(jsx: JSX.Element) {
  return {
    user: userEvent.setup(),
    ...render(jsx),
  };
}

it("should render", async () => {
  const { user } = setup(<Button />);
  await user.click(screen.getByRole("button"));
});
```

## 调试测试

以下工具对调试测试非常有用：

- [screen.debug()](https://testing-library.com/docs/queries/about/#screendebug) - 不带参数调用时，打印文档 DOM 树的可读表示；带参数时，打印指定节点的 DOM 树。内部使用 `console.log` 将输出记录到终端。
- [Testing Playground](https://testing-playground.com/) - 一个交互式沙箱，可用于测试哪些查询适用于特定的 HTML 元素。
- [prettyDOM logRoles](https://testing-library.com/docs/dom-testing-library/api-debugging/#prettydom) - 一个工具函数，打印给定 DOM 树中所有隐式 ARIA 角色。

## 测试 Select 组件

这里以 [OrgRolePicker](https://github.com/grafana/grafana/blob/38863844e7ac72c7756038a1097f89632f9985ff/public/app/features/admin/OrgRolePicker.tsx) 组件为例。该组件本质上是 `Select` 组件的封装，带有自己的一组选项。

```tsx
import { OrgRole } from "@grafana/data";
import { Select } from "@grafana/ui";

interface Props {
  value: OrgRole;
  disabled?: boolean;
  "aria-label"?: string;
  inputId?: string;
  onChange: (role: OrgRole) => void;
  autoFocus?: boolean;
  width?: number | "auto";
}

const options = Object.keys(OrgRole).map((key) => ({ label: key, value: key }));

export function OrgRolePicker({
  value,
  onChange,
  "aria-label": ariaLabel,
  inputId,
  autoFocus,
  ...restProps
}: Props) {
  return (
    <Select
      inputId={inputId}
      value={value}
      options={options}
      onChange={(val) => onChange(val.value as OrgRole)}
      placeholder="Choose role..."
      aria-label={ariaLabel}
      autoFocus={autoFocus}
      {...restProps}
    />
  );
}
```

### 查询 Select 组件

推荐通过标签来查询 `Select` 组件。添加一个 `label` 元素，并通过 `htmlFor` 属性匹配对应的 `inputId`。或者，也可以在 `Select` 上指定 `aria-label`。

```tsx
describe("OrgRolePicker", () => {
  it("should render the picker", () => {
    setup(
      <>
        <label htmlFor={"role-picker"}>Role picker</label>
        <OrgRolePicker
          value={OrgRole.Admin}
          inputId={"role-picker"}
          onChange={() => {}}
        />
      </>,
    );
    expect(
      screen.getByRole("combobox", { name: "Role picker" }),
    ).toBeInTheDocument();
  });
});
```

### 测试选项的正确显示

有时需要验证 `Select` 组件是否显示了正确的选项。在这种情况下，最佳做法是点击 `Select` 组件，然后使用 `*ByText` 查询匹配目标选项。

```tsx
it('should have an "Editor" option', async () => {
  const { user } = setup(
    <>
      <label htmlFor={"role-picker"}>Role picker</label>
      <OrgRolePicker
        value={OrgRole.Admin}
        inputId={"role-picker"}
        onChange={() => {}}
      />
    </>,
  );
  await user.click(screen.getByRole("combobox", { name: "Role picker" }));
  expect(screen.getByText("Editor")).toBeInTheDocument();
});
```

### 选择选项

为了简化从 `Select` 组件中选择选项的过程，提供了 `selectOptionInTest` 工具函数。该函数是对 [react-select-event](https://testing-library.com/docs/ecosystem-react-select-event/) 包的封装。

```tsx
it("should select an option", async () => {
  const mockOnChange = jest.fn();
  setup(
    <>
      <label htmlFor={"role-picker"}>Role picker</label>
      <OrgRolePicker
        value={OrgRole.Admin}
        inputId={"role-picker"}
        onChange={mockOnChange}
      />
    </>,
  );
  await selectOptionInTest(
    screen.getByRole("combobox", { name: "Role picker" }),
    "Viewer",
  );
  expect(mockOnChange).toHaveBeenCalledWith("Viewer");
});
```

## 模拟对象和函数

### 模拟 `window` 对象及其方法

模拟 `window` 对象的推荐方式是使用 [Jest spies](https://jestjs.io/docs/jest-object)。Jest 的 spy 函数提供了内置的 mock 恢复机制，无需手动保存 `window` 对象的引用。

```tsx
let windowSpy: jest.SpyInstance;

beforeAll(() => {
  windowSpy = jest.spyOn(window, "location", "get");
});

afterAll(() => {
  windowSpy.mockRestore();
});

it("should test with window", function () {
  windowSpy.mockImplementation(() => ({
    href: "www.example.com",
  }));
  expect(window.location.href).toBe("www.example.com");
});
```

### 模拟 getBackendSrv()

使用 `getBackendSrv()` 函数向 Grafana 后端发送 HTTP 请求。可以使用 `jest.mock` 方法模拟该函数。

```tsx
jest.mock("@grafana/runtime", () => ({
  ...jest.requireActual("@grafana/runtime"),
  getBackendSrv: () => ({
    post: postMock,
  }),
}));
```

#### 为 AsyncSelect 模拟 getBackendSrv

使用 `AsyncSelect` 组件异步加载选项。该组件通常依赖 `getBackendSrv` 来加载选项。

以下是对 [OrgPicker](https://github.com/grafana/grafana/blob/38863844e7ac72c7756038a1097f89632f9985ff/public/app/core/components/Select/OrgPicker.tsx) 组件的测试，该组件底层使用了 `AsyncSelect`：

```tsx
import { screen, render } from "@testing-library/react";
import userEvent from "@testing-library/user-event";

import { OrgPicker } from "./OrgPicker";

jest.mock("@grafana/runtime", () => ({
  ...jest.requireActual("@grafana/runtime"),
  getBackendSrv: () => ({
    get: () =>
      Promise.resolve([
        { name: "Org 1", id: 0 },
        { name: "Org 2", id: 1 },
      ]),
  }),
}));

function setup(jsx: JSX.Element) {
  return {
    user: userEvent.setup(),
    ...render(jsx),
  };
}

describe("OrgPicker", () => {
  it("should render", async () => {
    render(
      <>
        <label htmlFor={"picker"}>Org picker</label>
        <OrgPicker onSelected={() => {}} inputId={"picker"} />
      </>,
    );

    expect(
      await screen.findByRole("combobox", { name: "Org picker" }),
    ).toBeInTheDocument();
  });

  it("should have the options", async () => {
    const { user } = setup(
      <>
        <label htmlFor={"picker"}>Org picker</label>
        <OrgPicker onSelected={() => {}} inputId={"picker"} />
      </>,
    );
    await user.click(
      await screen.findByRole("combobox", { name: "Org picker" }),
    );
    expect(screen.getByText("Org 1")).toBeInTheDocument();
    expect(screen.getByText("Org 2")).toBeInTheDocument();
  });
});
```
