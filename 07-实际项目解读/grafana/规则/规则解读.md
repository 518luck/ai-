这个文件本质上是 Grafana 仓库给 AI agent 的开发规则手册，告诉 AI 在这个项目里怎么改代码、怎么跑测试、哪些地方不能乱动。
核心内容

- Grafana 是一个监控和可观测性平台。
- 后端主要是 Go。
- 前端主要是 TypeScript/React。
- 仓库是 monorepo，前端用 Yarn workspaces，后端用 Go workspaces。
  它要求 AI 做事的原则
- 遵循周围已有代码风格。
- 新功能必须写测试。
- 改动要聚焦，不能过度设计。
- 前端和后端改动要拆成不同 PR。
- 注意安全问题：XSS、SQL injection、command injection。
- 注释只能解释“为什么”，不要解释显而易见的代码。
- 代码注释里不能放 Slack、GitHub、Jira 等链接。
  最重要的限制
- 在运行 git push 之前，必须先获得人类明确批准。
- 即使任务里写了 “Open a PR”，也不代表 AI 可以直接 push。
- AI 应该先展示改动摘要，等人确认。
  它列出的常用命令
- 构建和运行：
- make run
- make build-backend
- yarn start
- yarn build
- 测试：
- go test -run TestName ...
- make test-go-unit
- make test-go-integration
- yarn test path/to/file
- yarn e2e:playwright ...
- lint 和格式化：
- make lint-go
- yarn lint
- yarn lint:fix
- yarn prettier:write
- yarn typecheck
- 代码生成：
- make gen-go
- make gen-cue
- make swagger-gen
- make gen-feature-toggles
- make update-workspace
  它说明了后端结构
- pkg/api/：HTTP API handlers 和 routes
- pkg/services/：业务逻辑
- pkg/server/：服务器初始化和 Wire DI
- pkg/tsdb/：时序数据库查询后端
- pkg/plugins/：插件系统
- pkg/infra/：日志、指标、数据库访问
- pkg/middleware/：HTTP middleware
- pkg/setting/：配置管理
  后端关键规则是：业务逻辑应该放在 pkg/services/<domain>/，不要塞进 API handlers。
  它说明了前端结构
- public/app/core/：共享服务、组件、工具函数
- public/app/features/：按业务领域组织的功能代码
- public/app/plugins/：内置插件
- public/app/types/：TypeScript 类型
- public/app/store/：Redux store 配置
  前端关键规则是：使用 Redux Toolkit、函数组件、hooks、useStyles2、RTK Query、React Testing Library。
  它特别提醒的工程规则
- 改后端 service init 后要跑 make gen-go。
- 改 kinds/ 里的 schema 后要跑 make gen-cue。
- 改 feature toggles 后要跑 make gen-feature-toggles。
- 新增 Go module 后要跑 make update-workspace。
- 数据库迁移在 pkg/services/sqlstore/migrations/。
- 后端测试支持通过 SHARD / SHARDS 做 CI 分片。
  Cursor Cloud 部分说明
- 环境需要 Node.js v24.x、Go 1.25.7、Yarn 4.11.0、GCC。
- make run 会启动后端，默认地址 localhost:3000，账号密码 admin/admin。
- yarn start 会启动前端开发服务器。
- 默认不需要外部数据库，因为 Grafana 使用嵌入式 SQLite。
- 前端测试默认 watch，要用 yarn jest --no-watch 或 --watchAll=false。
- 后端某些包测试编译慢，建议用 -run TestName 定向测试。
  一句话总结
  这个文件告诉 AI：在 Grafana 这种大型 Go + TypeScript 项目里，必须按已有架构、小步改动、补测试、跑生成命令、遵守前后端边界，并且 push 前必须让人类 review。
