## 五、各项目独特亮点

### Kibana 独有

- 插件架构约束（server entry 懒加载模式）
- 模块分组与可见性规则（shared vs private）
- DevEx 反馈机制：`echo "..." | scripts/devex_feedback.sh`
- 国际化（i18n）专项指导
- CI 说明章节
- 指定 UI 框架（`@elastic/eui` + Emotion）

### OpenCode 独有

- Good/Bad 代码对比示例块（四个中唯一）
- "ALWAYS USE PARALLEL TOOLS" AI 行为指令
- "Prefer automation" 自主操作授权
- 默认分支明确声明（`dev` 而非 `main`）
- "快乐路径"组织模式（主函数在上，辅助函数在下）
- Effect schema 偏好：`Schema.UnknownFromJsonString` 优于手动 `JSON.parse`

### Sentry 独有

- 层级式 AGENTS.md 系统 + 上下文感知加载（最成熟的设计）
- 权威来源声明（AGENTS.md 优先于 CLAUDE.md/Cursor rules）
- Feature Flags 完整工作流（注册 → 检查 → 测试 → 灰度发布）
- 客户信息安全禁令
- Skills 系统（`.agents/skills/` 目录）
- Git worktrees 支持
- 项目结构目录树图
- 命令区分 AI 自动化 vs 人工场景

### WP Calypso 独有

- 多客户端架构详解（Calypso、Jetpack Cloud、A8C for Agencies、Dashboard）
- 技术栈差异说明（Redux + Webpack vs TanStack Query + TanStack Router）
- Pre-PR checks 独立章节
- PR 模板引用 + 具体格式规则（Linear ID、禁止人名、禁止 wordpress.com URL）
- Dashboard 客户端"Self-contained"声明

## 七、核心发现

四份文件虽然风格各异（从 96 行的精简导航到 258 行的全面手册），但都围绕同一个核心目标：**让 AI 代理能够安全、高效、准确地在该项目中工作。**

它们共同覆盖了 **"项目认知 → 环境搭建 → 代码编写 → 质量保证 → PR 提交"** 的完整开发流程，每个环节都提供了 **可直接执行的命令** 和 **明确的边界约束**。

最佳实践是取各项目之长，形成 **"精确命令 + Good/Bad 示例 + 层级引导 + AI 行为指令"** 的完整模板。
